## Design Context

### Users
Linux socket course project reviewers and students use this interface to verify account, forum, private chat, group chat, file, notification, and backup features quickly in a browser.

### Brand Personality
Clear, reliable, restrained. The interface should feel like a practical community product rather than a decorative demo.

### Aesthetic Direction
Use a light, information-dense community workspace inspired by mainstream forums and messaging products: forum lists should scan like Discourse/Zhihu, and chat should read like WeChat/Feishu with separate conversations, visible hierarchy, and calm status feedback.

### Design Principles
- Put real workflows first: login, conversation switching, posting, replying, and notifications must be easy to find.
- Keep dense information readable with strong spacing rhythm, stable panels, and restrained color.
- Make chat messages feel conversational with avatars, sender metadata, and clear self/peer alignment.
- Make forum content feel like a reading surface with clear title, author, excerpt, replies, and attachment states.
- Preserve the existing backend protocol and avoid UI changes that create untestable behavior.
