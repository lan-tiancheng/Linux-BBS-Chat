# 项目报告初稿

## 1. 项目概述

本项目实现一个基于 Linux Socket 的多客户端聊天与 BBS 系统，
支持注册登录、群聊、私聊、文件传输和基础持久化。

## 2. 总体架构

- `server` 负责监听、认证、广播、日志和文件接收
- `client` 负责交互输入、消息接收和下载保存
- `protocol` 负责通用收发、长行处理和端口解析
- `user` 负责账号存储和校验
- `chat` 负责聊天日志
- `file_transfer` 负责上传、下载和本地保存

## 3. 成员分工

### 成员 A

负责后端通信、群聊、私聊、在线用户和文件传输核心逻辑。

### 成员 B

负责前端/终端交互、BBS 功能和用户操作流程。

### 成员 C

负责数据存储结构、目录设计、构建脚本、README、报告和最终打包。

## 4. 数据与目录设计

- `data/`：用户、帖子、回复、文件索引
- `logs/`：聊天日志
- `uploads/`：服务端上传文件
- `downloads/`：客户端下载文件
- `backup/`：备份和归档预留

当前已实现：

- `users.db`：用户账号
- `posts.db`：帖子索引
- `replies.db`：回复索引
- `files.db`：文件记录
- `chat.log`：聊天日志

其中用户、聊天和文件传输逻辑已经接入，帖子和回复的业务展示层
仍需要继续完善。

## 5. 工程组织

工程采用 `src/` + `include/` 的方式组织源码，
`Makefile` 提供 `all`、`clean`、`install`、`uninstall` 和 `test`。

## 6. 已完成内容

- TCP 长连接服务端和客户端
- 用户注册与登录
- 在线列表和群聊/私聊
- 文件上传和下载
- 聊天日志记录
- 持久化目录与文件约定
- 帖子、回复、文件记录的存储结构初稿
- 多客户端和断线重连测试

## 7. 主要问题与解决办法

- 问题：多客户端并发访问账号状态
  - 解决：使用互斥锁保护在线状态和用户列表
- 问题：文件传输需要区分文本命令和二进制内容
  - 解决：命令行和文件帧分离，按声明大小读写
- 问题：路径和目录可能不存在
  - 解决：把运行时目录纳入工程约定，并在初始化阶段创建
- 问题：工程层和业务层分散
  - 解决：新增统一的 `storage` 层，把路径和备份逻辑收口

## 8. 扩展讨论

- 后续可以继续做 BBS 帖子、回复和附件索引
- 可以把文本存储升级成 SQLite 或轻量数据库
- 可以补充更完整的异常恢复和备份策略

## 9. 结论

当前项目已经具备基础通信和文件传输能力，后续重点是
完善 BBS 模块、补齐提交材料并完成最终联调。
